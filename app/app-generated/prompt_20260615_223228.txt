SECTION 4 — CATEGORY CLASSIFICATION SYSTEM

════════════════════════════════════════════════════════
Every narrative must be classified across THREE independent dimensions. Each dimension answers a different question about the narrative.
── DIMENSION 1: SECTOR (What part of crypto is involved?) ──
AI              — Artificial intelligence tokens, AI agent platforms, AI infrastructure

DeFi            — Decentralized finance: lending, borrowing, yield, trading platforms

L2              — Layer 2 scaling networks built on top of Ethereum

RWA             — Real-world assets being tokenized (real estate, bonds, commodities)

Stablecoins     — Dollar-pegged tokens (USDC, USDT, DAI) and stablecoin protocols

Infrastructure  — Core blockchain infrastructure, bridges, oracles, node providers

Gaming          — Blockchain gaming, virtual worlds, in-game economies

NFT             — Non-fungible tokens, digital collectibles, NFT marketplaces

Memecoin        — Meme-driven tokens with primarily speculative value

Derivatives     — Futures, perpetual contracts, options, and leveraged positions

Other           — Does not fit any sector above
── DIMENSION 2: ACTOR (Who is moving the money?) ──
Institutional     — Known funds, ETFs, institutional custodians, corporate treasuries

SmartMoney        — On-chain wallets with strong historical track records (not necessarily institutions)

RetailSpeculation — Broad retail participation, community-driven, social-media-driven flows

Protocol          — Automated protocol activity: liquidations, rebalancing, governance execution
── DIMENSION 3: BEHAVIOR (What pattern is occurring?) ──
Accumulation    — Sustained buying or building of positions over multiple days

Distribution    — Selling or unwinding of positions, moving tokens toward exchanges

Rotation        — Capital leaving one sector or token to enter another

CrossChain      — Movement of assets between different blockchain networks via bridges

Liquidation     — Forced selling due to margin calls or protocol-triggered position closures

Bridging        — Capital moving specifically to access a new chain's ecosystem

Staking         — Locking tokens to earn yield or secure a network

Deployment      — Active putting-to-work of capital into protocols (lending, liquidity provision)
── CATEGORY SELECTION RULES ────────────────────────────
RULE A: Assign exactly ONE value per dimension. Do not leave any dimension blank.

RULE B: Use the dimension that best answers each question independently:

• SECTOR   = what kind of crypto is this about?

• ACTOR    = who is doing this?

• BEHAVIOR = what are they doing with their money?

RULE C: Never create two narratives that differ only by category label for the same underlying capital movement. If two signals share the same ACTOR + BEHAVIOR, merge them into one narrative.

RULE D: If a narrative involves multiple sectors, pick the dominant one. Do not split into two narratives.

RULE E: "Other" in SECTOR is a last resort — exhaust all other options first.
── Anti-duplication checklist ───────────────────────────
Before creating a new narrative, ask:

Does an existing narrative share the same ACTOR + BEHAVIOR? → Merge, do not create new.
Does an existing narrative share the same SECTOR + BEHAVIOR? → Merge unless actors are clearly distinct.
Is the only difference the token involved? → Add to top_tokens of the existing narrative, do not create new.

SECTION 5 — EVOLUTION SCENARIOS

════════════════════════════════════════════════════════
SCENARIO A — CONTINUING / ACCELERATING

Condition: A theme in Section 3 matches an existing narrative_id in Section 2.

Action: Reuse the same narrative_id. Set status=CONTINUING or ACCELERATING.

Add new observations to key_evidence — do not repeat prior evidence.

Update confidence based on fresh signals. Update top_tokens if changed.
SCENARIO B — NEW NARRATIVE

Condition: A theme in Section 3 has signal_count ≥ 2 AND no match in Section 2.

Action: Create a new narrative_id. Set status=NEW.

The narrative must represent a real-world investor story, not a token cluster.
SCENARIO C — STABLE / STAGNATING

Condition: An existing narrative_id has NO matching signals in Section 3,

AND hours_since_update < 72.

Action: STILL RETURN IT. Set status=STABLE, reduce confidence slightly.

This keeps valid multi-day themes alive through quiet data cycles.
Evidence → confidence mapping (STRICTLY enforce):

• signal_count ≥ 3 + acceleration        → confidence 0.75–0.85

• signal_count ≥ 3, no acceleration      → confidence 0.70–0.79

• signal_count = 2                       → confidence 0.55–0.69

• signal_count = 1, volume > $50M        → confidence 0.40–0.54

• signal_count = 1, bridge only          → confidence max 0.50

• Single-day data only                   → cap at 0.74, note in data_caveat
DO NOT return narratives with confidence_score < 0.40.

DO NOT fabricate signal evidence not present in Section 3.

SECTION 6 — WRITING STYLE (investor briefing rules)

════════════════════════════════════════════════════════
You are writing a retail investor briefing, not a data system output.
RULES:

Translate every signal into plain-English meaning:

NEVER write:  "whale_usd increased in token_flows metric"

ALWAYS write: "Large investors moved about $X into [token]"
Signal translations:

smart_money       → "experienced large investors" or "wallets with a strong track record"

bridge_activity   → "money moving between blockchains" or "capital rotating into [chain]"

whale_transactions→ "very large holders shifting funds" (typically $500K or more per move)

stablecoin_mint   → "fresh buying power entering the market" or "new dollars arriving in crypto"
Every narrative must answer four questions in plain English:

• What is happening right now?

• Who is moving money (what type of investor)?

• Why does it matter for the broader market?

• What could happen next?
Context all dollar amounts with a size anchor:

"$1.2B" → "$1.2 billion — roughly the size of a mid-sized investment fund"

"$50M"  → "about $50 million, a significant single-day institutional move"
No jargon without an immediate explanation in parentheses on first use:

OK: "decentralized exchanges (platforms where people trade crypto directly)"

NOT OK: "DEX volume spiked"
FORBIDDEN in output text fields (implications, summary, retail_considerations):

"signal_count" • "confluence" • "dex concentration" • "net outflow" • "on-chain"

"L1" • "L2" • "CEX" • "net flow" • "token_flows" • "whale_usd"

If you need to express these concepts, use the plain-English equivalents above.
plain_english_summary: exactly 2 sentences, texting-a-friend style:

"Big money wallets are quietly moving funds into Ethereum. This kind of positioning

has historically preceded price moves, but there's no guarantee it will this time."
retail_considerations: exactly 3 parts:

(1) What this means for regular investors in plain English

(2) One specific thing to watch as a signal of continuation or reversal

(3) A clear risk reminder: "On-chain movements are early signals that can reverse quickly —

this is not financial advice."

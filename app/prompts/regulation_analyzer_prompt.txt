  SOURCE VALIDATION REQUIREMENTS (MANDATORY)

  Step 0A — Validate Every URL

  Before performing research:

  For each source:

  * Confirm successful retrieval.
  * Confirm the page contains relevant content.
  * Confirm the page is not:

    * 404
    * Page not found
    * Redirect loop
    * Empty page
    * Unrelated content
    * Parked domain

  If any of the above occur:

  * Mark the URL invalid.
  * Do not cite it.
  * Do not include it in output.

  Step 0B — Discover Updated URLs

  If a listed URL is invalid:

  * Search the publisher's website.
  * Find the most current equivalent page.
  * Prefer:

    * Policy trackers
    * Regulatory trackers
    * Publications
    * Client alerts
    * Insights
    * Regulatory updates

  Replace obsolete URLs with current working URLs.

  Record replacements in metadata.replaced_sources.

  Step 0C — Source Quality Gate

  A source may only be used if:

  1. The page loads successfully.
  2. The content is relevant to digital assets, crypto, tokenisation, blockchain, stablecoins, AML/KYC, CBDCs, securities regulation, banking regulation, payments, or financial regulation.
  3. The publication date is visible.
  4. The publisher is identifiable.
  5. The publication falls within the lookback window OR is a tracker discussing developments within the lookback window.

  If any requirement fails:

  Exclude the source.
